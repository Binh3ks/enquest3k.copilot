Based on a thorough re-examination of the code and the Blueprint, here is the definitive structure of a standard week, using Week 20 as the "golden standard."

read.js (for Read & Explore station):

Purpose: Provides the main reading passage for the ReadingExplore component.
Schema: An object with title, image_url, content_en (with bolded keywords), content_vi, and an array of comprehension_questions.
Blueprint Match: Aligns with "Tab Read & Explore," requiring a reading passage, 10 bold words, and comprehension checks.
vocab.js (for New Words station):

Purpose: Defines the 10 core vocabulary words for the week. This data is also used by other stations like Word Match.
Schema: An array of 10 word objects, each containing id, word, definition_en, definition_vi, sentence_en, sentence_vi, image_url, audio_word, and audio_def.
Blueprint Match: Aligns with "Tab New Words," specifying 10 words with visuals and audio.
word_power.js (for Word Power station):

Purpose: Defines advanced vocabulary (collocations, synonyms, idioms).
Schema: An array of word objects, similar to vocab.js but focused on phrases, with examples.
Blueprint Match: Aligns with "Tab Word Power," which details a phased approach to collocations, synonyms, and idioms.
grammar.js (for Grammar station):

Purpose: Provides multiple-choice or fill-in-the-blank grammar questions.
Schema: An array of question objects, each with a question, an array of options, and the correct_answer.
Blueprint Match: Aligns with "Tab Grammar," specifying 20 questions per week.
logic_lab.js (for Logic Lab station):

Purpose: Presents math and logic problems with a story context.
Schema: An array of problem objects, each with a question_en, question_vi, image_url (optional), answer, and hint.
Blueprint Match: Aligns with "Tab Logic Lab," which requires contextualized math/logic problems.
ask_ai.js (for Ask AI station):

Purpose: A structured speaking exercise where the user asks a specific question based on a given context.
Schema: An array of prompt objects, each with a context_en, context_vi, an array of acceptable answers (the questions the user should ask), and a hint.
Blueprint Match: Aligns with "Tab Ask AI," requiring context-based prompts to encourage speaking.
dictation.js (for Dictation station):

Purpose: Provides sentences for the user to listen to and type out.
Schema: An array of sentence objects, each with an id, sentence, and audio_url.
Blueprint Match: Aligns with "Tab Dictation," using sentences from the main reading passage.
explore.js (for Explore station):

Purpose: A non-fiction reading passage focused on critical thinking.
Schema: An object with title_en, image_url, content_en (with bolded words), content_vi, an array of check_questions, and a final open-ended question for critical response.
Blueprint Match: Aligns with "Tab Explore," which is a higher-level reading comprehension task.
writing.js (for Writing Challenge station):

Purpose: Provides a prompt for a written response.
Schema: An object containing the prompt_en, prompt_vi, a model_sentence, and a word_bank.
Blueprint Match: Aligns with "Tab Writing Challenge," requiring a specific prompt, model sentences, and a word bank.
daily_watch.js (for Daily Watch station):

Purpose: Contains links to curated YouTube videos.
Schema: An array of video objects, each with a title, video_id, and description.
Blueprint Match: Aligns with "Tab Daily Watch," which specifies using a whitelist of YouTube channels.
mindmap.js (for Mindmap Speaking station):

Purpose: Provides sentence stems and branches for a structured speaking exercise.
Schema: An object with centerStems (an array of starting phrases) and branchLabels (an object mapping each stem to an array of possible endings).
Blueprint Match: This is a new feature not yet in the Blueprint.
word_match.js:

Purpose: Placeholder file. The WordMatch component dynamically uses data from vocab.js.
Schema: export default null; or a placeholder object.
shadowing.js:

Purpose: Placeholder file. The Shadowing component dynamically uses the reading passage and audio from read.js.
Schema: export default null;
Detailed Descriptions of New Features
Here are the detailed descriptions of the three new features you asked for. You can add these to your Blueprint document.

1. Feature: AI Tutor
Nature: A global, persistent AI assistant accessible from anywhere in the app, separate from the weekly station flow. It is a comprehensive, multi-functional tool for advanced practice.
Component: AITutor.jsx
Data Source: Does not use a dedicated weekly data file. It dynamically pulls context from the current week's data (e.g., vocabulary from new_words) and the master syllabus_database.js.
Core Functionality: The AI Tutor is organized into five distinct tabs:
Chat: An open-ended conversational agent. Users can engage in free-form chat or select from predefined role-playing scenarios (e.g., "At the Doctor," "Ordering Food") that become available as they progress through the weeks.
Pronunciation: An isolated practice tool where users can select a word from the current week's vocabulary list and have their pronunciation scored by the AI.
Story Builder: A collaborative writing tool where the user and the AI build a story together, sentence by sentence. The AI provides guiding questions and hints to help the user continue the narrative.
Debate: A formal debate feature where the user can choose a topic and argue a point against the AI. The AI acts as the opponent, providing counter-arguments.
Quiz: A dynamic quiz generator that creates questions on Vocabulary, Math, or Science based on the current week's syllabus and content.
2. Feature: Mindmap Speaking
Nature: A structured speaking and production activity, integrated as a standard weekly station.
Component: MindMapSpeaking.jsx
Data Source: Requires a mindmap.js file in each week's data folder.
Core Functionality: The interface presents a "mind map" with a central theme.
The user is shown a "sentence stem" (e.g., "In the old castle, there was ___.").
Several "branches" appear, each containing a phrase to complete the sentence (e.g., "a secret passage," "a huge dining hall").
The user selects a branch, which forms a complete sentence.
The user then records themselves speaking the full sentence aloud. The primary goal is to practice forming and speaking grammatically correct, contextually relevant sentences.
3. Feature: Game Hub
Nature: A self-contained station dedicated to a collection of mini-games for review and engagement.
Component: GameHub.jsx
Data Source: Does not require a dedicated weekly data file. The games and their logic are hardcoded within the component itself.
Core Functionality: The Game Hub contains a variety of different game types that may use vocabulary or concepts from the current week. The games are designed to be fast, fun, and reinforce learning in a low-pressure environment. Examples of games found within the component include word scrambles, quick-fire quizzes, and other interactive challenges.
